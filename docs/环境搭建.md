### 1. 安装 Node.js 的“正确姿势”：NVM
在 Linux 里，我们从不直接安装 Node.js，而是用 **NVM**（Node Version Manager）。它可以让你随时切换 Node 版本（比如项目 A 要用 v18，项目 B 要用 v20）。

在你的 Ubuntu 窗口输入：
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```
安装完后，**重启你的终端**（关掉再开），然后安装最新的长期支持版 Node：
```bash
nvm install --lts
node -v  # 看到版本号说明成功了
```

### 2. 让“黑窗口”变漂亮的秘密：Oh My Zsh
前端开发者通常不喜欢默认的 Bash，而是用 **Zsh**。配合 **Oh My Zsh**，你的终端可以显示当前的 Git 分支、语法高亮、自动补全。

1.  安装 Zsh：`sudo apt install zsh -y`
2.  安装 Oh My Zsh：
    ```bash
    sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
    ```
    *你会发现黑窗口瞬间变酷了，心情好了写代码效率也高。*

### 3. 最强生产力组合：WSL + VS Code
这是你以后每天的工作状态。请按照这个步骤尝试：

1.  **在 Linux 里建个项目文件夹**：
    ```bash
    mkdir -p ~/projects/learning-js
    cd ~/projects/learning-js
    ```
2.  **用 VS Code 敲开 Linux 的门**：
    输入命令：
    ```bash
    code .
    ```
3.  **神奇的事情发生了**：Windows 里的 VS Code 会启动，但你会发现它的左下角显示着 **"WSL: Ubuntu"**。
    *   这时候，你在 VS Code 里的**插件**、**终端**、**文件系统**全是 Linux 的。
    *   创建一个 `index.js`，写一句 `console.log("Hello JS from Linux!")`。
    *   在 VS Code 内部的终端输入 `node index.js`，直接运行。

### 4. 为什么这对前端开发更“高效”？

*   **npm install 极快**：前端项目的 `node_modules` 包含成千上万个小文件。在 WSL2 的 Linux 文件系统里安装插件的速度，比在 Windows 下快 5-10 倍。
*   **路径兼容性**：Linux 区分大小写，Windows 不区分。很多前端包在 Windows 下部署到服务器（通常是 Linux）时会报错，但在 WSL 里开发能保证“开发环境 = 生产环境”。
*   **原生支持 Git**：所有的 Git 钩子（Git Hooks）和自动化脚本在 Linux 下运行得最完美。

在 Linux (WSL) 里使用 Git 是非常丝滑的，因为 Git 原生就是为 Linux 设计的。在 WSL 里使用 Git，主要分为 **配置、连接 GitHub/Gitee、以及在 VS Code 里操作** 三步。

---

### 第一步：检查并配置你的身份
Ubuntu 通常预装了 Git。先检查一下：
```bash
git --version
```

然后告诉 Git 你是谁（这会出现在你的代码提交记录里）：
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

---

### 第二步：生成 SSH 密钥（高手必做）
如果你想把代码推送到 GitHub 或码云，**不要用账号密码连接**，那样每次都要输密码，而且不安全。我们要生成一对“密钥”，像门禁卡一样自动刷卡。

1.  **生成密钥**：
    ```bash
    ssh-keygen -t ed25519 -C "你的邮箱"
    ```
    *一路狂按回车（不要设置密码），直到出现一串方框字符图。*

2.  **获取公钥**：
    ```bash
    cat ~/.ssh/id_ed25519.pub
    ```
    *你会看到一串以 `ssh-ed25519` 开头的长字符串。**把它全部复制下来**。*

3.  **把公钥交给 GitHub**：
    *   打开 GitHub -> 点击右上角头像 -> **Settings**。
    *   找到左侧的 **SSH and GPG keys**。
    *   点击 **New SSH key**，把刚才复制的东西粘贴进去，取个名字（比如 `My-WSL`），点 **Add**。

---

### 第三步：在 VS Code 里快乐使用
既然你已经在 VS Code 里连接了 WSL，使用 Git 就有两种方式：

#### 1. 图形界面（新手最爱）
*   在 VS Code 左侧侧边栏，点击第三个图标（看起来像一条分叉的路，快捷键 `Ctrl + Shift + G`）。
*   当你修改了代码（比如 `hello.js`），你会在这里看到文件变动。
*   点击文件旁边的 **`+`**（代表 `git add`）。
*   在上面的框里输入提交信息（比如 "feat: my first js code"），点 **Commit**（代表 `git commit`）。

#### 2. 命令行（高手必备）
在 VS Code 底部的终端（Zsh）里，这是你的“每日三部曲”：
```bash
git add .              # 1. 把所有改动放进暂存区
git commit -m "描述"    # 2. 存进本地仓库（存档）
git push               # 3. 推送到云端（GitHub）
```

### 第四步：实战演练（建一个仓库）
现在给你的 `learning-js` 项目创建一个 Git 存档：

1.  在终端进入项目文件夹：
    ```bash
    cd ~/projects/learning-js
    ```
2.  初始化仓库：
    ```bash
    git init
    ```
3.  你会发现你的 Zsh 终端提示符变了（Oh My Zsh 会显示当前在 `master` 分支），这就是它好用的地方！
4.  提交第一次存档：
    ```bash
    git add .
    git commit -m "initial commit"
    ```

---

### 💡 一个关于 WSL 的“避坑”指南
**绝对不要在 Windows 的 PowerShell 里对 WSL 的文件夹运行 Git 命令！**
*   虽然可行，但因为跨文件系统（Windows Git 去读 Linux 文件），速度会慢得像蜗牛。
*   **规则：** 在 Ubuntu 里的代码，就在 Ubuntu 的终端（或者连接了 WSL 的 VS Code 终端）里跑 Git。
